🎙️ VoIP Streaming Engine - Architecture Summary

This document details our hybrid error correction and recovery framework designed to handle real-time audio over unstable networks.

Core Design & System Mechanisms

Rather than relying on resource-intensive FEC block matrices or simple duplication, this system implements a Parity-Based Forward Error Correction coupled with a Feedback ARQ (NACK) Loop.

  • Parity Forward Path: Even-numbered packets are sent normally (164 bytes). Odd-numbered packets are extended (324 bytes) to contain the previous frame's payload. This ensures that single drops are recovered immediately on arrival of the next packet.
  • Under-the-Hood Framing: To save protocol overhead, no headers are added. The receiver automatically distinguishes packet types by inspecting the size of the UDP datagram returned by recvfrom().
  • Targeted NACK Retransmissions: If consecutive drops occur, the receiver detects sequence gaps. It fires a lightweight 4-byte NACK request back to the sender, which retransmits the missing data frame.
